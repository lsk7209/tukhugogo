## P4 이도현 — "페이지는 문서가 아니라 제품이라며, 정작 제품이 없다"

**지적 1. v1.1에 'task'가 없다.**
프로젝트 원칙은 "페이지=제품, 사용자가 실제 작업을 완료해야 함"인데, v1.1의 코어 페이지(랭킹·트렌드·기술 가이드)는 **읽고 끝나는 문서**다. "AI 특허 순위" 검색자는 답 5초 보고 이탈한다. 체류·재방문·완료할 task가 없다. 이건 SCA 방어("제품화")의 핵심이 비어있다는 뜻이고, zero-click과도 직결된다.

**지적 2. 인터랙티브 요소가 한 줄짜리 빈칸.**
v1은 Phase 4에 "인터랙티브 요소"를 한 줄 적고 명세가 없다. 이게 이 사이트의 **유일한 진짜 차별점**인데(단순 통계는 KIPRIS·Google Patents가 무료 제공) 비어있으면 안 된다.

**지적 3. 검색의도 매핑이 없다.**
키워드마다 의도가 다른데(정보형/탐색형/거래형) v1.1은 페이지 타입을 의도에 안 맞췄다. "특허 출원 방법"(거래형, 변리사 리드)과 "AI 특허 순위"(정보형, GEO)는 완전히 다른 페이지여야 한다.

**제안 — 인터랙티브 도구 3종 (task completion):**
1. **기업 특허 비교기** — 2~3개 기업 선택 → 기술분야별 출원·등록률 나란히 차트. (탐색형 의도, 재방문·체류 유발, 공유 가능)
2. **기술분야 출원 타임라인 탐색기** — 연도 슬라이더로 분야별 출원 변화 + 상위 기업 변동.
3. **관심 기업/기술 추적** (경량, 로컬) — 즐겨찾기 → 재방문 동기.
→ 이 도구들이 매트릭스 데이터를 **소비 가능한 task**로 바꾼다. 셀 페이지는 도구의 데이터 백본이 되어 noindex여도 가치 발생.

**제안 — 검색의도 레이어:**
키워드를 정보형/탐색형/거래형으로 분류 → 정보형=GEO 인용 블록, 탐색형=인터랙티브 도구, 거래형=가이드+제휴 CTA.

## P5 장세훈 — "단위경제가 안 나올 수 있고, 더 나은 카드가 손에 있다" (가장 아픈 지적)

**지적 1. 냉정한 RPM 현실.**
- 일반인 트래픽 = 정보형 = zero-click(AI Overview가 답) → PV 낮음.
- 전문가 트래픽 = 광고 클릭 거의 안 함 → CTR·RPM 낮음.
- 즉 **PV도 RPM도 둘 다 낮은 버티컬.** 애드센스 단독은 단위경제가 위험하다.

**지적 2. 비용은 실재한다.**

| 비용 항목 | 추정 (※가정, 검증 필요) |
|---|---|
| 데이터 | 무료 한도 초과 시 KIPRISPlus 유료 구간 |
| AI 보강 | Gemini Flash로 수천 페이지 enrich (1회+갱신) |
| 유지보수 | 월1 갱신 + 출원인 정규화 관리(수동 큐레이션 지속) + 등록상태 정합성 |
| 시간 | Phase 0.5~6 빌드 + 운영 |

**지적 3. 손익 추정 (보수적, 전부 가정).**
```
수익 = 월 트래픽 × CTR × CPC  (특허 버티컬: CTR·낮음 가정)
→ 일반트래픽 약함 + zero-click + 낮은 CTR = 애드센스 수익 박할 가능성 큼
```
> **권고:** 가공의 RPM 숫자 말고, **bojo24의 실제 RPM을 프록시로** 단위경제를 먼저 계산하라. 라이브 사이트 실측이 가장 정확하다. 그 숫자로 "셀당 빌드/운영 비용 회수되나"를 판정.

**지적 4. 기회비용 — 손에 더 나은 카드가 있다.**
- **bojo24**: 이미 라이브. 출산지원금 트래픽 → 태아보험 CPA 확장 여지(수요·전환의도 명확).
- **연봉지도(Concept B)**: 수요가 직종-전국 키워드에 명확히 존재(검증됨).
- **PassRo(자격증)**: 수요·API 검증 진행 중, 일반 수요 명확.
- **특허맵**: 수요 불확실 + 라이선스 불확실 + 단위경제 불확실. **3중 불확실.**
→ 냉정히, 특허맵은 현재 포트폴리오에서 **가장 불확실성이 높은 카드**다. 끌린다는 이유로 더 확실한 카드보다 앞세우는 게 맞나?

**제안 — 수익모델 재정의 (애드센스 단독 탈피):**
1. **가이드형 미드테일 → 애드센스** (정보 소비형, 일반인 키워드)
2. **거래형 키워드 → 제휴 리드** (특허출원 대행·IP컨설팅·상표등록 서비스 CPA/리드) ← 의도매치 시 전환율 높음
3. **랭킹/트렌드 → GEO 인용** (브랜드·권위, 직접 수익 ❌)
4. (선택) 정제 데이터 → B2B 프리미엄
→ CPA 의도매치 원칙(메모리): 거래형 페이지 구조를 제휴 오퍼 세그먼트에 맞춰 설계.

## 5인 합의 & 충돌

**신규 합의:**
4. **제품(인터랙티브 도구)이 SCA 방어·차별화·재방문의 핵심** → 도구 3종을 코어 산출물로 승격. (P4)
5. **애드센스 단독 단위경제는 위험** → 수익모델 3분화(애드센스/제휴리드/GEO). (P5)
6. **bojo24 실측 RPM으로 손익 선검증** 후 진행 여부 재확인. (P5)

**충돌 (Limo 결정 필요):**
- P5: "Gate① 수요가 약하게 나오면, 특허맵은 후순위로 미루고 더 확실한 카드(연봉지도/PassRo)에 자원을 몰자."
- P1·P4: "인터랙티브 도구 + 제휴리드로 재정의하면 수요가 약해도 거래형·도구형으로 살릴 여지 있다."
→ **분기 조건:** Gate① 수요 결과 + bojo24 RPM 프록시 손익. 둘 다 약하면 P5 손, 하나라도 강하면 P1·P4 손.

## 개선안 v1.2 — 변경된 결정 (v1.1 위에 추가)

| 항목 | v1.1 | **v1.2 (개선)** |
|---|---|---|
| 인터랙티브 도구 | "한 줄" | **3종 명세화(비교기·타임라인·추적), 코어 산출물로 승격** |
| 검색의도 | 미분류 | **정보/탐색/거래 3분류 → 페이지 타입 매핑** |
| 수익모델 | 하이브리드(모호) | **3분화: 애드센스(가이드)/제휴리드(거래형)/GEO(랭킹) + 선택 B2B** |
| 단위경제 | 없음 | **bojo24 실측 RPM 프록시로 손익 선검증 (하드 판단)** |
| 포트폴리오 우선순위 | 미논의 | **Gate①+손익 결과로 다른 컨셉 대비 우선순위 재결정** |
| 매트릭스 셀 | noindex 롱테일 | **인터랙티브 도구의 데이터 백본으로 재정의 (페이지 노출 아니어도 가치)** |

### v1.2 페이지·제품 구조
```
제품층(차별화·재방문):  기업 비교기 · 타임라인 탐색기 · 관심 추적   ← 신규 핵심
코어(index, 수요검증):  기술분야 가이드 + 랭킹 + 트렌드
거래형(제휴 CTA):       "특허 출원 방법/비용" 등 → IP서비스 리드
허브(index):           기업허브 + 기술허브
백본(noindex):          매트릭스 셀 = 도구·집계의 데이터 소스
```

## 최종 평결 (5인)

**조건부 GO — 단, 3중 게이트 + 우선순위 의식적 결정.**

- v1.2로 재정의(인터랙티브 도구 + 수익 3분화)하면 컨셉 자체는 살릴 만하다.
- 그러나 P5 지적이 무겁다: **수요·라이선스·단위경제 3중 불확실**이고, 손에 더 확실한 카드(bojo24/연봉지도/PassRo)가 있다.
- 따라서 평결은 "GO 가능하되, **아래 3개를 통과하고 + 다른 컨셉보다 앞세울 이유를 스스로 납득**한 뒤에만":
  1. **Gate ① 수요검증** (DataLab) — 🔴면 후순위
  2. **라이선스 서면확인** (KIPRISPlus) — 🔴면 보류
  3. **손익 선검증** (bojo24 RPM 프록시) — 회수 불가면 수익모델 제휴리드로 강제 전환
